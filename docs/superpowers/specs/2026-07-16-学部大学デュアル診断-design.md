# 学部/大学デュアル診断＋大学ページ 設計

## 目的
現行の学部単位3ゾーン診断（engine_v2）に加え、同じ入力・同じエンジンで「大学単位」の診断も見られるようにする。さらに大学全体ページを新設し、大学↔学部を相互に行き来できるようにする。

## 確定した方針（ユーザー承認済み）
- **大学診断のランキング**: engine_v2の学部結果を大学単位に集約。各大学は「最も適性スコアの高い学部（＝代表学部）」で代表させ、その fitScore / zone / reasons を大学の値とする。
- **モード選択**: 結果画面上部の切替タブ ［🎓 学部で見る / 🏫 大学で見る］。診断実行は1回、同じ `state.lastResult` から両ビューを描画。デフォルトは学部。
- **一覧**: 「全大学を見る」を大学一覧（地域別）に変更。大学→大学全体ページ→学部→学部詳細。
- データ・エンジンは data_v2 + engine_v2 の1系統のまま。お気に入りは学部単位を維持。

## 画面
- 既存: home / diag-* / result / list / detail(学部詳細) / favs / loading
- **新規: `uni-detail`（大学全体ページ）** — `#uniDetailBody` を持つ section

## コンポーネント設計

### 1. 大学集約（純ロジック）
`aggregateByUni(result)`:
- 入力: `state.lastResult`（engine_v2の zones.{challenge,match,safe} + ungrouped）
- 各 zone 内の学部項目を uni_id でグルーピング
- 各大学の代表 = そのゾーン内でfitScore最大の学部項目
- 戻り値: `{ challenge:[uniItem], match:[uniItem], safe:[uniItem], ungrouped:[uniItem] }`
- `uniItem` 形: `{ uni_id, uni_name, repDept:{dept_id,dept_name}, hensachi:{min,max}(代表学部), fitScore, reasons, zone, deptCount(そのゾーンでの学部数) }`
- 同一大学が複数ゾーンに跨る場合は各ゾーンにその大学の代表を出す（ゾーンは学部単位の実力帯なので妥当）。

### 2. 結果ビュー切替
- result画面上部に `#resultViewTabs`（学部/大学）。
- `state.resultView = 'dept' | 'uni'`（既定 'dept'）。
- `renderResults()` を分岐: dept → 既存の学部3ゾーン描画。uni → `aggregateByUni` 後に大学カードで3ゾーン描画。
- サマリ（換算偏差値・Holland・条件タグ・免責）は共通で上部に出す。

### 3. 大学カード `uniCard(uniItem)`
- 大学名 / 代表学部ヒント（例: 「最も合う: 経営学部（適性52）」）/ 代表偏差値帯 / 適性バッジ / 理由チップ / 「大学ページ →」ボタン（`data-uni="uni_id"`）。
- クリック → `renderUniPage(uni_id)` → go('uni-detail')。

### 4. 大学全体ページ `renderUniPage(uniId)`
- 大学ヘッダ: 名称・地域バッジ・区分・所在地・偏差値帯（全学部のmin〜max）。
- 大学情報（data_v2 uni）: feature / access_from_shiga_min・commute / tuition・living_cost / shingaku_jisseki・career_center / official_links。
- **学部リスト**: その大学の全学部を偏差値降順で。各行: 学部名・偏差値・キャンパス・「詳しく見る →」（`data-detail="dept_id"`）→ 学部詳細。

### 5. 学部詳細 → 大学ページ導線
- `renderDetail()` の末尾に「🏫 この大学の全体を見る →」ボタン（`data-uni="uni_id"`）→ `renderUniPage` → go('uni-detail')。

### 6. 一覧の大学化 `renderList()`
- data_v2.universities を地域別（REGION_ORDER）に、地域内は代表偏差値（全学部mid最大）降順。
- 大学カード（簡易: 名称・区分・偏差値帯・学部数）→ 大学ページ。検索は uni.name / name_short / 学部名を対象。

## イベント委譲
- 既存の document 委譲に `data-uni` を追加: `renderUniPage(id); go('uni-detail')`。
- `data-detail`（学部詳細）・`data-fav`（お気に入り）は現状維持。

## お気に入り
- 学部単位のまま（`favorites-v2ui`、dept_idキー）。大学ページの学部リストにも❤️は付けてよいが必須ではない（YAGNI: 今回は学部カード/結果のみで可）。

## テスト
- `aggregateByUni` を node で単体テスト（bridge/engine同様に純関数として `work/agg_v2.js` に切り出し、`work/test_agg_v2.js`）。
  - 複数学部を持つ大学が代表学部（最大fit）で1件に集約されること
  - ゾーンごとに集約されること
- 実機E2E: 学部/大学タブ切替、大学カード→大学ページ→学部詳細、学部詳細→大学ページ、大学一覧→大学ページ。

## 非対象（YAGNI）
- v1の旧大学スコアリング復活はしない（集約で代替）。
- 大学単位のお気に入り比較表は復活しない。
